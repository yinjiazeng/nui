路由

页面不刷新实现页面切换效果，用于开发单页应用

使用模块

在页面中创建元素，并包含href属性，并且创建一个空元素作为渲染容器

下载 路由组件作为依赖项引入到模块中，并创建实例，查看 demo

选项参数

# path

类型：{String}

默认值：null

必填：✔

说明：路径匹配规则，冒号为动态参数名，url匹配上该规则才会触发onInit回调

# container

类型：{Selector}

默认值：null

必填：✔

说明：设置渲染容器，实例创建后会生成（<div class="nui-router-wrapper"></div>）作为container子元素，该元素就是路由模版填充容器

# target

类型：{Selector}

默认值：null

说明：给元素绑定路由，元素必须包含href属性，点击该元素触发路由跳转

# data

类型：{Object}

默认值：{}

说明：模版渲染数据，默认值包含path（路由地址），url（包含动态参数路由地址），query（路由查询参数，“?”后面的参数），params（路由参数，包含查询参数与动态参数），默认值不可手动修改。

# template

类型：{String, Object}

默认值：''

说明：渲染模版，当值为对象（多个模版）时，必须存在名称为main模版作为主模版，模版中可以直接访问data中数据

# entry

类型：{Boolean}

默认值：false

说明：是否设置为入口页面，页面中只能设置一个入口；当url不满足path规则时，会自动跳转到入口页面

# wrapper

类型：{Boolean, Selector}

默认值：false

说明：值为false时，所有设置该值的路由实例都将共享一个容器，每次加载都会触发onInit；当值为true时，实例会独享一个容器，仅在第一次加载时时会触发onInit；值为选择器时同true，只不过该元素必须是container的子元素且class值必须包含nui-router-wrapper。

# level

类型：{Number}

默认值：2

说明：当匹配规则path中包含参数时才会启用，取值分别为0、1、2。当值为0时url必须满足匹配条件，参数不能少；当值为1时允许url少参数，但不能无参数；当值为2时允许url不传参数

# events

类型：{Object}

默认值：null

说明：用于绑定事件，用法参考代理事件

# onBefore

类型：{Function}

默认值：null

说明：路由地址改变之前触发，若返回false将不会切换地址

参数：change {Function} 调用后会强行切换路由地址

# onData

类型：{Function}

默认值：null

参数：data {Anything} 接收数据

说明：路由地址切换后，在渲染模版之前触发，用于接收路由跳转时传递的数据

# onChange

类型：{Function}

默认值：null

说明：路由地址切换后，在渲染模版之前触发，函数返回false时则不会执行渲染模版，不会触发onInit和onAfter

# onRender

类型：{Function}

默认值：null

说明：渲染模版后触发，每次调用实例的render方法都会触发

# onInit

类型：{Function}

默认值：null

必填：✔

说明：渲染模版后触发，当选项参数wrapper为true时仅会触发一次

# onAfter

类型：{Function}

默认值：null

说明：渲染模版后触发，不论选项参数wrapper是何值都触发

# 说明

在选项参数函数中可以通过this.element访问到页面容器的jquery实例（类名为nui-router-wrapper的元素），绑定事件必须代理为该元素。万万不可将事件代理到页面加载就存在的元素上，比如documeng或者body，因为每次切换路由都会触发回调导致事件被多次绑定！

实例方法

# render

类型：{Function}

说明：渲染模版，调用后会触发onRender回调，该方法一般在onInit中异步请求数据后调用

静态方法

# config

类型：{Function}

参数：option {String, Object} 选项参数集合或者参数名

参数：value {Anything} 参数值，option为string类型时才会使用

说明：全局设置默认选项参数，需在调用组件前设置方可生效

# start

类型：{Function}

说明：初始化了路由跳转到主入口页面

# location

类型：{Function}

参数：url {String} 跳转地址

参数：data {Anything} 跳转传递数据，跳转后的页面通过onData函数可接收

参数：render {Boolean} 是否重新渲染目标页面

说明：路由跳转，跳转的地址必须匹配已设置的路由规则，否则不会跳转

# forward

类型：{Function}

参数：index {Number} 要访问的url在历史记录的位置

说明：前进，等同history.forward，可无需手动调用该方法，在元素添加类属性nui-router-forward即可

# back

类型：{Function}

参数：index {Number} 要访问的url在历史记录的位置

说明：返回，等同history.back，可无需手动调用该方法，在元素添加类属性nui-router-back即可

# destroy

类型：{Function}

参数：id {Number, String} 实例id或者选项参数id

说明：销毁实例，不传id则对所有实例操作
